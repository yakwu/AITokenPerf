# AITokenPerf UX 重构设计

> 日期：2026-04-11
> 状态：待审批

## 1. 背景与目标

### 当前问题
1. **数据查看不直观**：测试结果散落在历史表格里，缺少一眼判断「这个站点行不行」的能力
2. **配置流程割裂**：单次测试和定时任务各自有配置入口，配置（站点连接信息）和测试参数混在一起
3. **页面职责不清**：测试页功能过于复杂，配置页只是辅助，缺少以站点为核心的管理视角

### 设计目标
- **配置是独立实体，测试只是引用它**：以「目标站点」为核心组织所有功能
- **数据直观化**：多指标综合展示、按模型拆分、趋势迷你图、错误分布
- **异常可溯源**：所有异常指标、告警、错误分布均可点击，跳转到对应的站点详情或当次测试结果，快速定位根因
- **流程打通**：页面间有快捷跳转，减少操作步数
- **定时任务优先**：日常以定时监控为主，手动测试为辅

## 2. 页面结构

### 导航（4 个主 Tab）

```
概览 | 目标站点 | 历史与对比 | 定时任务    [运行指示器] [头像 ▾]
                                         ↑ 浮层      ↑ 设置
                                                     ↑ 模型管理（管理员）
                                                     ↑ 用户管理（管理员）
                                                     ↑ 退出
```

### 页面变化对照

| 当前 | 新版 |
|------|------|
| Dashboard | 保留为「概览」首页，增强异常感知 |
| 配置页 | 升级为「目标站点」主页，卡片化 + 快速测试 |
| 测试页 | 拆掉。测试融入站点详情页的「测试」Tab |
| 历史页 | 增强为「历史与对比」，加入趋势对比 |
| 定时任务（测试页内） | 独立为导航 Tab + 站点详情页内 Tab，双入口 |
| 模型管理（导航 Tab） | 移入用户下拉菜单，仅管理员可见 |

## 3. 概览页（首页 `/`）

### 布局
- **顶部 5 卡片**：目标站点数 / 活跃定时任务数 / 今日测试数 / 整体成功率 / 覆盖模型数
- **左列：站点状态列表**
  - 每站一行：健康灯（绿/灰/红） + 站点名 + base_url + 最新 TTFT / Token·s⁻¹ / 成功率
  - 异常站点红色背景高亮，未测试站点灰色
  - 点击跳转目标站点详情页
- **右列上：异常告警**
  - 退化事件列表：站点名 + 退化描述 + 错误类型标签（如 `503 × 12`、`Timeout × 5`）
  - **点击站点名 → 跳转站点详情页（历史趋势 Tab）**
  - **点击错误标签 → 跳转最近一次异常测试的结果详情**（展开错误明细、HTTP 状态码、具体报错）
  - **点击退化描述 → 跳转历史与对比页，自动筛选该站点，定位退化时间点**
- **右列下：最近活动**
  - 时间线：时间 + ✓/✗ + 站点名 + 模型 + 成功率 + 来源（定时/手动）
  - **点击任意一条 → 跳转该次测试的结果详情**
  - **✗ 标记的记录优先排在前面**，引导用户关注异常

### 数据来源
- 定时轮询刷新（复用现有批量轮询机制）
- 所有数据可点击跳转

## 4. 目标站点页 `/sites`

### 顶部操作栏
- 站点计数
- 搜索框（按名称/URL 搜索）
- 状态筛选 pills：全部 / 健康 / 异常 / 未测试
- 「+ 新建站点」按钮

### 双列卡片网格
每张站点卡片包含：

1. **头部**：健康灯 + 站点名 + base_url + 状态标签 + 上次测试时间
2. **按模型拆分的指标表**：每模型一行，列 = 模型名 / TTFT / TPOT / Token·s⁻¹ / 成功率 / 7天趋势迷你图
3. **错误类型分布**：
   - 正常站点：错误标签横排（如 `429 × 3`、`503 × 1`），标注「近 N 次请求」
   - 异常站点：分布条 + 退化警告（如「成功率较昨天下降 35%」）
   - **点击错误标签 → 跳转历史页，自动筛选该站点 + 该错误类型，定位出错的具体请求**
   - **点击退化警告 → 跳转站点详情的历史趋势 Tab，自动聚焦退化时间窗口**
4. **底部**：模型数 / 定时任务状态 + 「一键测试」按钮 + 「详情 →」按钮

### 卡片状态
- **健康**（成功率 ≥ 95%）：正常边框，绿色指标
- **异常**（成功率 < 95% 或指标退化）：红色边框，红色指标 + 退化提示
- **未测试**：虚线边框，灰色，「尚无测试数据」

### 响应式
- 桌面：双列网格
- 移动端：单列

## 5. 站点详情页 `/sites/:id`

面包屑导航：`← 目标站点 / 站点名`

### 4 个内部 Tab

#### Tab 1：测试

**参数配置区**（3 列布局）：
- 模型选择：combobox（输入框 + 下拉列表，复用现有模型列表，支持搜索和自定义模型 ID）
- 模式选择：Burst / Sustained 二选一
- 并发级别：预设值 pill + 自定义

**高级参数**（折叠行）：
- 请求数 / 超时 / Max Tokens / 提示词，内联展示当前值，点击展开编辑

**操作按钮**：开始测试 / Dry Run

**结果区**（两列模型卡片）：
每张模型卡片：
- 上层：4 个核心指标大字（TTFT P50 / TPOT P50 / Token·s⁻¹ / 成功率）
- 下层：补充指标紧凑横排（TTFT P95/P99 / TPOT P99 / E2E / Output Tokens / 错误信息），自然换行
- 「详情 →」入口：展开完整百分位表、错误明细、Token 时间线

#### Tab 2：定时任务

**任务列表**（表格）：
- 列：任务名 / 状态（运行中/已暂停） / 频率 / 模型 / 并发 / 上次结果（成功率+时间） / 下次执行 / 操作（暂停/恢复/立即执行/编辑/删除）

**新建任务**（内联表单）：
- 只需填 3 项：任务名称 / 频率（下拉选择 + 自定义 Cron） / 模型选择
- 并发、模式、超时等参数**自动继承站点的测试配置**
- 提示：「参数继承自站点默认配置，如需自定义点击高级设置」

#### Tab 3：历史趋势

- 指标切换 pills：TTFT / TPOT / Token·s⁻¹ / 成功率
- 时间范围：1h / 24h / 7d / 30d
- 折线图：按模型分线（不同颜色），异常点标注

#### Tab 4：配置

- 站点名称 / Base URL / API Key（密码显示）/ 绑定模型（标签式，可添加/删除）
- 操作：保存配置 / 连通性测试 / 删除站点

## 6. 定时任务页 `/tasks`

### 全局管理页
- 筛选：全部 / 运行中 / 已暂停 / 异常
- 表格列：站点名（可点击跳转站点详情） / 任务名 / 状态 / 频率 / 模型 / 并发 / 上次结果 / 下次执行 / 操作
- 「+ 新建任务」按钮（需选择目标站点后进入站点详情的定时任务 Tab）

### 顶部浮层指示器
- 位置：导航栏右侧，用户头像左边
- 显示：绿点 + `N 运行中` 文字
- 点击展开浮层：
  - 所有运行中任务列表：任务名 / 站点名 / 频率 / 最近成功率 + 时间
  - 异常任务红色高亮
  - 底部「查看全部定时任务 →」跳转 Tab 页

## 7. 历史与对比页 `/history`

### 列表视图
**筛选栏**：
- 时间范围：24h / 7d / 30d / 全部
- 站点下拉 / 模型下拉 / 来源下拉（定时/手动）

**多选对比**：
- 每行 checkbox，选中后顶部出现已选标签栏
- 标签栏：「已选 N 条」+ 各记录标签（可移除）+ 「对比」按钮 + 「取消」

**表格列**：
时间 / 站点 / 模型 / 并发 / 来源标签 / TTFT P50 / TPOT P50 / Token·s⁻¹ / 成功率 / 操作（详情/重测）
- 异常记录红色背景
- 定时任务的连续执行自动分组（分组头显示任务名 + 站点 + 频率）
- 「重测」快捷跳转到对应站点的测试 Tab

### 对比视图（点击「对比」后切换）
- **对比表格**：每列一条记录，行 = 指标。最优值绿色加粗，差值标百分比（如 `+637%`）
- **对比趋势图**：选中记录的指标随时间变化折线图

## 9. 异常溯源跳转矩阵

所有异常相关的 UI 元素均可点击跳转，定位根因：

| 触发位置 | 可点击元素 | 跳转目标 |
|----------|-----------|----------|
| 概览页 - 异常告警 | 站点名 | 站点详情页 → 历史趋势 Tab |
| 概览页 - 异常告警 | 错误标签（如 `503 × 12`） | 该站点最近一次异常测试的结果详情（展开错误明细） |
| 概览页 - 异常告警 | 退化描述 | 历史与对比页，自动筛选该站点，定位退化时间点 |
| 概览页 - 最近活动 | ✗ 记录行 | 该次测试的结果详情 |
| 概览页 - 最近活动 | ✓ 记录行 | 该次测试的结果详情 |
| 概览页 - 站点状态 | 站点行 | 站点详情页 |
| 目标站点卡片 | 错误标签 | 历史页，筛选该站点 + 错误类型 |
| 目标站点卡片 | 退化警告 | 站点详情 → 历史趋势 Tab，聚焦退化窗口 |
| 目标站点卡片 | 指标值（红色/黄色） | 站点详情 → 历史趋势 Tab，聚焦该指标 |
| 顶部运行指示器 | 异常任务行 | 站点详情 → 定时任务 Tab |
| 顶部运行指示器 | 「查看全部」 | `/tasks` 定时任务页 |
| 历史列表 | 异常记录行 | 该次测试结果详情（展开错误明细 + 错误分布） |
| 历史列表 | 「重测」按钮 | 站点详情 → 测试 Tab，预填该次参数 |
| 定时任务表格 | 站点名 | 站点详情页 |
| 定时任务表格 | 上次结果（异常） | 该次测试结果详情 |

## 11. 实施策略

按以下顺序分阶段实施，每阶段可独立交付：

### Phase 1：页面结构重组
- 新增路由和导航（4 Tab + 用户菜单调整）
- 目标站点页：基础卡片列表（不含趋势迷你图和错误分布）
- 站点详情页：4 Tab 框架 + 配置 Tab（迁移现有 ProfileView 功能）
- 移除旧测试页路由

### Phase 2：测试功能迁移
- 站点详情「测试 Tab」：迁移 TestView 的测试功能
- 模型选择改为 combobox（复用现有模型列表 API）
- 测试结果区：两列模型卡片布局

### Phase 3：定时任务
- 站点详情「定时任务 Tab」
- 定时任务全局页 `/tasks`
- 顶部浮层运行指示器

### Phase 4：数据可视化增强
- 目标站点卡片：加入趋势迷你图 + 错误类型分布
- 站点详情「历史趋势 Tab」
- 概览页重构

### Phase 5：历史与对比增强
- 历史与对比页新布局
- 多选对比 + 对比视图
