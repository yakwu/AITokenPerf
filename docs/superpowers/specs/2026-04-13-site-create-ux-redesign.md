# 站点创建流程 UX 整改

## 问题

当前「新建站点」跳转到 /config 页面（ProfileView），用户旅程混乱：

1. 跳转到连接配置管理页，用户不知道发生了什么
2. 点已有配置会复用，而非新建
3. 缺少配置名称时点保存没反应（toast 被定时任务通知挡住，且无表单内联校验）
4. 新建模式下保存按钮是橙色圆球，语义不清
5. 保存成功后停留在 /config 页面，不跳转到站点页

## 方案：SitesView 弹窗创建

点击「新建站点」→ Modal 弹窗 → 一步填写所有配置 → 创建后跳到站点详情页。

### 弹窗布局

```
┌─────────────────────────────────────┐
│  新建站点                      ✕     │
├─────────────────────────────────────┤
│  站点名称  [                        ]│
│                                     │
│  目标地址  [                        ]│
│  API Key   [ ••••••••        ] [显示]│
│  模型厂商  [ ▼ Azure OpenAI      ]  │
│                                     │
│  模型选择  [搜索或选择模型...    ▼]  │
│           ✓ gpt-4o  ✓ gpt-4o-mini  │
│                                     │
│            [ 创建站点 ]  [ 取消 ]    │
└─────────────────────────────────────┘
```

### 改动清单

#### 1. SitesView.vue — 新建站点弹窗

- 删除 `router.push('/config')` 跳转
- 新增创建站点弹窗（ModalOverlay），包含：
  - 站点名称（必填）
  - 目标地址 / Base URL（必填）
  - API Key（必填，带显示/隐藏切换）
  - 模型厂商 / Provider 下拉选择
  - 模型多选 combobox（选完站点后从 API 拉取模型列表）
- 表单内联校验：必填字段为空时，字段下方红色提示，不靠 toast
- 点击「创建站点」：
  - POST `/api/profiles/save` 一次性创建
  - 成功后 toast「站点已创建」+ 跳转 `/sites/{name}?tab=test`
- 按钮：常规矩形「创建站点」按钮，不用圆球

#### 2. 表单校验改进

- 所有必填字段为空时提交 → 第一个空字段聚焦 + 下方红色提示文字
- 不依赖 toast 做必填校验提示
- toast 仅用于成功/失败反馈，z-index 确保高于定时任务指示器

#### 3. ProfileView.vue 保留不变

- /config 页面作为高级配置管理入口保留
- 不影响现有编辑/删除/重命名功能
- 圆球按钮保留（只在高级管理页出现，主流程不再经过这里）

### 不做的事

- 不改 ProfileView 现有布局（那是高级管理入口）
- 不改 SiteConfigTab（站点详情页的配置编辑）
- 不改后端 API
